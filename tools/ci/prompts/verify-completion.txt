fully achieves every goal described in the planning document for this workstream.

This is NOT a review task. You are a **coding agent**. Your job is to **implement**
whatever is missing, write tests that prove the implementation works, and only report
completion once the branch is genuinely done.

## Your Task

### Step 1: Understand the Plan

Read the planning document (the workstream's system instructions tell you where it is).
Extract every goal, requirement, and deliverable.

### Step 2: Assess Current State

Use `git log ${BASE_BRANCH}..HEAD` and `git diff ${BASE_BRANCH}...HEAD` to understand
the full scope of changes already made on this branch. For each goal in the plan,
determine whether the implementation is complete, partial, or missing entirely.

### Step 3: Implement What Is Missing

For every goal that is incomplete or missing:

1. **Write the code.** Do not merely describe what needs to be done — implement it.
2. **Write meaningful tests.** Each goal must have tests that would **fail** if the
   implementation were removed or broken. Tests that pass regardless of the
   implementation do not count.
3. **Follow project conventions.** Test classes must extend `TestSuiteBase`, use
   `@TestDepth` for expensive tests, and include timeout parameters.
4. **Do not weaken existing tests.** Do not add `@Ignore`/`@Disabled` to pre-existing
   tests, remove assertions, add `try/catch` to swallow exceptions, or change expected
   values to match incorrect output.

If you are the first agent to work on this branch (only the plan document and supporting
files have been committed), this is expected — implement the entire plan.

Once you have completed all implementation work, verify the branch is done and post
your findings as a comment on the pull request (if one exists) or send a summary via
Slack:
