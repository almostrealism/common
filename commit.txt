Fix LICM correctness bug and add missing test coverage

The LICM (Loop-Invariant Code Motion) implementation in Repeated.java had a
correctness bug where non-declaration assignments with StaticReference
destinations were not tracked as variant names. This caused declarations
depending on those assigned variables to be incorrectly hoisted out of loops.

Root cause: StaticReference.getDependencies() returns empty, so
collectBaseVariantNamesRecursive never added the destination name to the
variant set when the assignment destination was a StaticReference.

Fix: Added explicit StaticReference name extraction in the non-declaration
assignment handling code in collectBaseVariantNamesRecursive.

Also adds two new LICM test cases from the plan document:
- nestedInnerLoopDeclarationNotHoisted: verifies inner-loop index
  references are NOT incorrectly hoisted while outer-loop invariants ARE
- declarationDependingOnAssignedVariableNotHoisted: verifies that
  declarations depending on variables assigned elsewhere in the loop body
  are NOT hoisted (the bug that was fixed)

Test results: 988 utils tests run at depth 0, 916 passed, 71 skipped,
1 unrelated timeout (WeightedSumIsolationRuntimeTest native compilation).
