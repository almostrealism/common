Fix renderCells NPE and restore Heap.stage() in pattern rendering

AudioScene: Initialize renderCells field to prevent NPE when
getPatternChannel() is called directly (outside getCells()). The field
was previously only initialized inside getCells(), causing a
NullPointerException when tests like frameRangeWithEffects invoke
getPatternChannel() independently.

PatternFeatures: Restore Heap.stage() scoping around note evaluation
in both cached and uncached render paths. The branch refactor of
render() into separate cached/uncached paths accidentally dropped the
Heap.stage() wrapper that the master code had, leading to
"Already allocated NativeRef" errors during native memory allocation.
