Wrap PatternFeatures.render() stream in Heap.stage() to prevent memory exhaustion

The getNoteDestinations() call inside render() triggers PackedCollection
allocations (offsetArg in ScaleTraversalStrategy.createRenderedNote()) that
were outside any Heap.stage() scope. This caused cascading "Already allocated
NativeRef" warnings and eventually "Hardware Memory max reached" errors in
tests with effects enabled (AudioSceneBufferConsolidationTest.effectsEnabledPerformance).

The fix wraps the entire stream pipeline in Heap.stage() so that all
allocations from getNoteDestinations() are lifecycle-managed. The existing
nested Heap.stage() calls around evaluate() continue to work correctly
since Heap uses a stack-based stage model.
