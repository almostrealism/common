Overview

The ar-chemistry module provides a complete model of the periodic table and chemical elements. It enables programmatic access to all 118 chemical elements with their atomic structures, electron configurations, and integration with the physics module for atom construction.

Complete Periodic Table

All 118 chemical elements from Hydrogen (1) to Oganesson (118), with accurate electron shell configurations and atomic properties.

Element Groups

Access elements by family (alkali metals, noble gases, halogens), period (row), group (column), or orbital block (s, p, d, f).

Alloy Creation

Probabilistic mixing of elements for material science applications. Define composition ratios and generate atoms based on those proportions.

Physics Integration

Seamless integration with the ar-physics module to construct Atom objects with proper electron shell configurations.

Elements and Periodic Table

Core Interfaces

Element

Represents a chemical element with atomic number and atom construction capability. Base interface for all 118 elements.

Atomic

Interface for substances that can construct Atom objects. Bridge between chemistry (elements) and physics (atoms).

Substance

Marker interface for all chemical substances. Enables participation in graph-based relationships and reaction networks.

PeriodicTable

Central access point for all elements. Provides static constants and methods for accessing elements by category.

Accessing Individual Elements

import org.almostrealism.chem.*;

Element Families

Alkali Metals

Li, Na, K, Rb, Cs, Fr - Group 1 (excluding H). Highly reactive, single valence electron.

Alkaline Earth Metals

Be, Mg, Ca, Sr, Ba, Ra - Group 2. Two valence electrons, important in biology.

Transition Metals

Groups 3-12 (d-block). Variable oxidation states, catalytic properties. Includes Fe, Cu, Au.

Halogens

F, Cl, Br, I, At, Ts - Group 17. Highly reactive nonmetals, seven valence electrons.

Noble Gases

He, Ne, Ar, Kr, Xe, Rn - Group 18. Chemically inert, complete valence shells.

Lanthanoids

La through Lu (57-71). Rare earth elements, f-block. Similar chemical properties.

Actinoids

Ac through Lr (89-103). f-block series. Many are radioactive (U, Pu).

Metalloids

B, Si, Ge, As, Sb, Te, At. Semiconductors, properties between metals and nonmetals.

Molecular Structures

Atomic Structure

Each element encapsulates its electron shell configuration, enabling accurate atom construction:

Element Classes

118 individual classes (Hydrogen, Carbon, etc.) implementing the Element interface with proper electron configurations.

Shell Configuration

Each element builds upon noble gas cores and adds electrons according to the aufbau principle.

Atom Construction

The construct() method creates Atom objects with the correct number of protons and electron shells.

Physics Module Integration

Atoms are created using the ar-physics module's Atom and Shell classes.

Electron Configuration Example

// Carbon: Atomic number 6
// Electron configuration: 1s2 2s2 2p2 or [He] 2s2 2p2
public class Carbon implements Element {

    public int getAtomicNumber() { return 6; }

    public Atom construct() {
        return new Atom(getAtomicNumber(), getShells());
    }

    private List<Shell> getShells() {
        ArrayList<Shell> shells = new ArrayList<>();
        // Reuse Helium's shell (1s2)
        shells.addAll(PeriodicTable.Helium.getShells());
        // Add second shell (2s2 2p2 = 4 electrons)
        shells.add(Shell.second(2, 2));
        return Collections.unmodifiableList(shells);
    }
}

Molecule Interface

The Molecule interface models chemical compounds as graphs where elements are nodes and bonds are edges:

// Molecule interface for future implementation
public interface Molecule extends Substance, Graph<Element> {

    // Get elements bonded to the specified element
    Collection<Element> neighbors(Element node);

    // Get total atom count in the molecule
    int countNodes();
}

// Example: Water molecule (conceptual)
// H-O-H with oxygen at center, 2 hydrogens connected
Molecule water = new WaterMolecule();
int atomCount = water.countNodes();  // 3 (2 H + 1 O)
Note: The Molecule interface is currently minimally implemented. Chemical bonding and molecular geometry are handled by the physics module. Future versions will provide complete molecular structure support.

Alloy (Material Mixtures)

The Alloy class enables probabilistic mixing of elements for material science applications:

Alloy

Probabilistic mixture of Atomic substances. When construct() is called, it randomly selects an element based on composition ratios.

ProbabilisticFactory

Parent class enabling genetic algorithm integration through the heredity module.

Gene-based Composition

Alloy compositions can be controlled by Genes, enabling evolution of material properties.

import org.almostrealism.chem.Alloy;
import java.util.Arrays;

// Bronze: 88% Copper, 12% Tin
Alloy bronze = new Alloy(
    Arrays.asList(PeriodicTable.Copper, PeriodicTable.Tin),
    0.88, 0.12
);

// Steel: 98% Iron, 1.5% Carbon, 0.5% Manganese
Alloy steel = new Alloy(
    Arrays.asList(PeriodicTable.Iron, PeriodicTable.Carbon, PeriodicTable.Manganese),
    0.98, 0.015, 0.005
);

// Brass: 70% Copper, 30% Zinc
Alloy brass = new Alloy(
    Arrays.asList(PeriodicTable.Copper, PeriodicTable.Zinc),
    0.70, 0.30
);

// Generate atoms from alloy (probabilistically selects element)
for (int i = 0; i < 1000; i++) {
    Atom atom = bronze.construct();
    // Approximately 880 atoms will be Cu, 120 will be Sn
}

Usage Examples

Basic Element Access

import org.almostrealism.chem.*;
import org.almostrealism.physics.Atom;

public class ElementDemo {
    public static void main(String[] args) {
        // Access elements
        Element carbon = PeriodicTable.Carbon;
        Element oxygen = PeriodicTable.Oxygen;
        Element hydrogen = PeriodicTable.Hydrogen;

        // Print atomic numbers
        System.out.println("Carbon: Z=" + carbon.getAtomicNumber());   // 6
        System.out.println("Oxygen: Z=" + oxygen.getAtomicNumber());   // 8
        System.out.println("Hydrogen: Z=" + hydrogen.getAtomicNumber()); // 1

        // Construct atoms
        Atom c = carbon.construct();
        Atom o = oxygen.construct();
        Atom h = hydrogen.construct();

        // Work with element groups
        List<Element> organics = PeriodicTable.organics();
        System.out.println("Organic elements: " + organics.size());
    }
}

Material Simulation

import org.almostrealism.chem.*;
import org.almostrealism.physics.Atom;
import java.util.*;

public class MaterialSimulation {

    public static void main(String[] args) {
        // Create a steel alloy
        Alloy steel = new Alloy(
            Arrays.asList(
                PeriodicTable.Iron,
                PeriodicTable.Carbon,
                PeriodicTable.Manganese
            ),
            0.98, 0.015, 0.005
        );

        // Count element distribution over 10000 samples
        Map<Integer, Integer> counts = new HashMap<>();

        for (int i = 0; i < 10000; i++) {
            Atom atom = steel.construct();
            int z = atom.getAtomicNumber();
            counts.merge(z, 1, Integer::sum);
        }

        // Print distribution
        System.out.println("Element distribution in steel sample:");
        System.out.println("Iron (26): " + counts.getOrDefault(26, 0));
        System.out.println("Carbon (6): " + counts.getOrDefault(6, 0));
        System.out.println("Manganese (25): " + counts.getOrDefault(25, 0));
    }
}

Troubleshooting

Common Issues

Issue: Cannot construct Atom from Element
Solution: Ensure ar-physics module is included as a dependency. The Atom and Shell classes come from the physics module.
Issue: Alloy composition doesn't sum to 1.0
Solution: Composition values are normalized internally. They don't need to sum to 1.0, just maintain the correct ratios.
Issue: Molecule methods throw UnsupportedOperationException
Solution: The Molecule interface is minimally implemented. Use concrete implementations or wait for future versions.
Issue: Missing element class
Solution: All 118 elements are included. Access via PeriodicTable.ElementName (e.g., PeriodicTable.Gold).
